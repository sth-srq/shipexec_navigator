📋 Executive Summary

ShipExec Navigator (branded Management Studio Pro) is a web-based application built to dramatically simplify the day-to-day administration of ShipExec shipping software. It replaces a collection of manual XML-editing workflows and disconnected utilities with a single, modern interface that any team member can use — no programming knowledge required.

At its core, Navigator connects directly to a live ShipExec environment and presents the entire company configuration as an interactive, explorable tree — making it possible to see every setting on one screen, make changes in place, and review exactly what will be updated before committing. A built-in AI Chatbot (powered by Azure OpenAI) can answer questions about ShipExec, search product documentation, and even suggest code improvements for client customizations.

Beyond configuration editing, Navigator provides bulk import and export for both shippers and users with intelligent column mapping, full audit history with undo capability for every change, application and security log viewing with CSV export, downloadable business-rule files, and a suite of standalone offline tools — including log viewers and a reusable JavaScript helper library generated through AI-driven analysis of existing client customizations.

The result is a faster, safer, and more transparent workflow for ShipExec support teams, implementation engineers, and client administrators alike.

1. AI Chatbot — General Questions & Rich Responses

The ShipExec Copilot is a conversational AI assistant available from the application's home page. Users can type natural-language questions about ShipExec — configuration concepts, field meanings, troubleshooting steps — and receive detailed answers instantly.

  • Responses are rendered as rich HTML: formatted text, bullet lists, tables, highlighted code snippets, and more — not plain text.
  • Full conversation history is maintained within each session, so follow-up questions retain context.
  • The chat can be cleared at any time to start a fresh conversation.
💡 The Copilot is always one click away from every page in the application via the top navigation bar.
📚

2. AI Knowledge of ShipExec Documentation (RAG)

The chatbot is enhanced with Retrieval-Augmented Generation (RAG) — a technique that lets the AI search an indexed library of ShipExec product documentation before answering a question.

  • When the 📚 RAG toggle is enabled, the AI automatically searches the document index for the most relevant passages related to the user's question.
  • The retrieved documentation is fed to the AI as additional context, resulting in answers that are grounded in real ShipExec reference material rather than solely relying on the AI model's general training data.
  • Users can toggle RAG on or off depending on whether they need documentation-backed answers or general assistance.
📖 The documentation index is built offline using a dedicated RAG Loader tool that processes ShipExec docs into searchable vector embeddings stored locally.

4. XML Viewer — In-Place Editing & Changes

Navigator doesn't just display configuration — it allows direct modifications within the tree. Users can edit values, add child entities, and delete items directly in the interface.

  • Inline editing — click on any value to change it. The system validates input using ShipExec's schema (enums, reference lookups, and field types).
  • Add & delete — context menus allow adding new shipper records, child elements, or removing entities.
  • Dirty-state indicator — an "unsaved" badge appears in the toolbar whenever changes exist, so nothing is accidentally overlooked.
  • View Changes — a dedicated button computes a full diff between the modified tree and the live server, producing a clear summary of every change before anything is applied.
  • Compare Profiles — side-by-side comparison of configuration profiles for auditing differences.
📋

5. Variances — Change Tracking, Storage & Undo

A variance is Navigator's term for a tracked change. Every modification — whether it's a field edit, a new entity, or a deletion — is captured as a variance entry and displayed in a real-time Changes Panel alongside the configuration tree.

How variances work:

  • Captured instantly — as soon as a value is modified, a variance is created showing the change type (edit, add, or delete), the path in the tree, and a description.
  • Before & after snapshots — each variance stores the original XML and the new XML so you can always see exactly what changed.
  • Stored in a database — when changes are applied to the live server, every variance is permanently recorded in the dbo.Variances table with a batch ID, company ID, user ID, timestamps, and full before/after data.
  • Batch grouping — all changes from a single "Apply" action share a common batch identifier, making it easy to see the complete set of changes from one operation.
  • Undo capability — individual variances can be undone directly from the Changes Panel. Undone entries are visually marked but retained in the history for full traceability.
  • Historical review — a history button loads all previously applied variances for the connected company, providing a complete audit trail of every change ever made through Navigator.
🔒 Variances provide a complete audit trail. Every change is traceable to a specific user, time, and batch — and every change can be reviewed or reversed.

6. AI-Assisted Configuration Updates

An AI Assistant panel is embedded directly within the Navigator page. When opened, it provides a chat interface that is aware of the currently loaded company configuration.

  • Users can ask the AI to make specific changes to the configuration — for example, "change the default carrier for site Chicago to FedEx" — and the AI will update the tree directly.
  • The AI can also answer questions about the loaded data — "which shippers have rate shopping enabled?" — using built-in XML analysis plugins.
  • Every AI-driven change is tracked as a variance, just like manual edits — same undo capability, same audit trail.
  • An Undo button in the chat panel allows reverting the most recent AI-driven changes instantly.
  • RAG document search is also available within the Navigator chat for documentation-backed answers in context.

7. Shipper Import with Dynamic Column Mapping

The Import Shippers dialog allows users to upload a CSV file containing shipper data and map its columns to ShipExec fields — with no fixed file format required.

  • Upload & preview — after selecting a CSV file, the system shows a preview table with all rows and column headers.
  • Dynamic column picking — each CSV column gets a dropdown selector listing every available ShipExec shipper field. Users choose which column maps to which field, handling any CSV layout without reformatting.
  • Custom data support — columns mapped as "Custom Data" get an additional key-name input for maximum flexibility.
  • Validation & feedback — row counts and mapping status are shown in real time. Import results report success and error counts.

8. Shipper Export

A single click on the Export Shippers toolbar button extracts all shipper records from the loaded company configuration and downloads them as a structured file. This is useful for backup, offline review, migration to another environment, or as a starting template for bulk re-import.

  • Same intuitive preview table and column-picking dropdowns as the shipper import.
  • Handles varying CSV formats without requiring a fixed template.
  • Results summary confirms how many users were created or updated.

10. User Export

From the Users tab, all user records for the connected company can be exported with a single action. The resulting file captures login names, roles, permissions, and associated metadata — ready for auditing, backup, or re-import.

11. Downloadable Server Business Rules (SBRs)

The Business Rules tab lists every Server Business Rule (SBR) associated with the connected company. Each entry displays its name, description, version, author, and the profiles it is assigned to.

  • SBR .dll files can be downloaded directly from the interface with a single click — no need to log into a server or use remote file access.
  • Useful for archival, local inspection, or transfer to another environment.

12. Client Business Rules — View & Modify

Client Business Rules (CBRs) are custom JavaScript files that control client-side behavior within ShipExec. Navigator provides a full view-and-edit experience for CBRs directly in the browser.

  • A sidebar lists every CBR for the company, showing name, version, and assigned profiles.
  • Selecting a CBR opens its JavaScript source code in a built-in editor.
  • Edits can be made directly in the editor and saved back — with cancel to revert to the original.
🧠

13. AI-Powered Code Improvement for CBRs

Beyond manual editing, Navigator can send a CBR's JavaScript to the AI along with the existing CBR Helper library and ask it to analyze and improve the code.

  • The Analyze button sends the current script and the CBRHelper.js utility library to the AI for review.
  • The AI identifies opportunities to use existing helper functions to simplify the code, reduce duplication, and follow best practices.
  • The suggested improved code is displayed in a read-only preview panel.
  • Users can Accept & Replace to adopt the AI's version, or Dismiss to keep the original.
🧠 This feature ensures that CBR code stays consistent and takes full advantage of the shared helper library, reducing maintenance effort across clients.
📋

14. Log & Security Log Export

The Logs tab within Navigator provides access to both Application logs and Security logs from the connected ShipExec environment.

  • Three viewing modes — Application only, Security only, or a combined view that merges both log types chronologically.
  • Date range filtering — specify start and end dates to narrow results.
  • Detailed columns — application logs show date, level, logger, message, server, and transaction ID. Security logs add user, action type, entity, and before/after values.
  • CSV export — a single button exports the current view as a CSV file for offline analysis, import into spreadsheets, or integration with the standalone log viewer tools.

A second standalone viewer — the Log Viewer — is purpose-built for the CSV log exports produced by Navigator itself (Application, Security, or Combined).

17. CBR Helper Library — Methodology & Approach

The CBR Helper (CBRHelper.js) is a downloadable JavaScript utility library that packages approximately 120 reusable helper functions for ShipExec CBR development. It was created through a systematic, AI-assisted analysis of real-world client implementations.

The five-step methodology:

  1. Collect all active CBRs — retrieve every Client Business Rule file from the UAT environment that is assigned to an active shipping profile, ensuring only current, in-use customizations are analyzed.
  2. Identify reusability topics — review each CBR and compile a master list of recurring functional topics: rate shopping, field validation, API calls, address book logic, UI manipulation, shipment field defaults, printing, LTL handling, and more.
  3. Analyze each file by topic — for every CBR, the AI examines the source code topic-by-topic, extracting concrete examples of reusable patterns, helper functions, and common logic.
  4. Condense into JavaScript objects — the extracted patterns are distilled into well-organized JavaScript objects grouped by category (Utilities, Logger, UI, API, Validation, Printing, LTL, Shipment, etc.), with consistent naming and documentation.
  5. Combine into a single CBRHelper — all category objects are merged into one global CBRHelper object, creating a single-file drop-in library that any CBR can reference for common operations instead of re-implementing them.
📥 The finished CBRHelper.js file is available for download from the Downloads page and can be dropped into any CBR project immediately.
